107 Etiquetas y Descripciones de HTML

Numero

Nombre de la Etiqueta

Descripcion
1 !doctype html Define que el documento esta bajo el estandar de HTML 5
2 html Representa la raiz de un documento HTML o XHTML. Todos los demas elementos deben ser descendientes de este elemento
3 head Representa una coleccion de metadatos acerca del documento, incluyendo enlaces a, o definiciones de, scripts y hojas de estilo.
4 title Define el titulo del documento, el cual se muestra en la barra de titulo del navegador o en las coletillas de pagina. Solamente puede contener texto y cualquier otra etiqueta contenida no sera interpretada
5 base Define la URL base para las URLs relativas en la pagina.
6 link Usada para enlazar JavaScript y CSS externos con el documento HTML actual.
7 meta Define los metadatos que no pueden ser definidos usando otro elemento HTML.
8 style Etiqueta de estilo usada para escribir CSS en linea.
9 script Define ya sea un script interno o un enlace hacia un script externo. El lenguaje de programacion es JavaScript
10 noscript Define un contenido alternativo a mostrar cuando el navegador no soporta scripting.
11 body Representa el contenido principal de un documento HTML. Solo hay un elemento body en un documento.
12 header Este elemento presenta informacion introductoria puede ser aplicado en diferentes secciones del documento, Tiene el proposito de contener la cabecera de una seccion etcâ€¦
13 section Define una seccion en un documento.
14 nav Define una seccion que solamente contiene enlaces de navegacion
15 article Define contenido autonomo que podria existir independientemente del resto del contenido.
16 aside Define algunos contenidos vagamente relacionados con el resto del contenido de la pagina. Si es removido, el contenido restante seguira teniendo sentido
17 h1,h2,h3,h4,h5,h6 Los elemento de cabecera implementan seis niveles de cabeceras de documentos; h1 es la de mayor y h6 es la de menor importancia. Un elemento de cabecera describe brevemente el tema de la seccion que introduce.
18 footer Define el pie de una pagina o seccion. Usualmente contiene un mensaje de derechos de autoria, algunos enlaces a informacion legal o direcciones para dar informacion de retroalimentacion.
19 address Define una seccion que contiene informacion de contacto.
20 main Define el contenido principal o importante en el documento. Solamente existe un elemento main en el documento.
21 p Define una parte que debe mostrarse como un parrafo.
22 hr Representa un quiebre tematico entre parrafos de una seccion o articulo o cualquier contenido.
23 pre Indica que su contenido esta preformateado y que este formato debe ser preservado.
24 blockquote Representa un contenido citado desde otra fuente.
25 ol Define una lista ordenada de articulos.
26 ul Define una lista de articulos sin orden.
27 li Define un articulo de una lista enumerada.
28 dl Define una lista de definiciones, es decir, una lista de terminos y sus definiciones asociadas.
29 dt Representa un termino definido por el siguiente dd.
30 dd Representa la definicion de los terminos listados antes que el.
31 figure Representa una figura ilustrada como parte del documento.
32 figcaption Representa la leyenda de una figura.
33 div Representa un contenedor generico sin ningun significado especial.
34 a Representa un hiperenlace, enlazando a otro recurso.
35 em Representa un texto enfatizado, como un acento de intensidad.
36 strong Representa un texto especialmente importante.
37 small Representa un comentario aparte, es decir, textos como un descargo de responsabilidad o una nota de derechos de autoria, que no son esenciales para la comprension del documento.
38 s Representa contenido que ya no es exacto o relevante.
39 cite Representa el titulo de una obra.
40 q Representa una cita textual inline.
41 dfn Representa un termino cuya definicion esta contenida en su contenido ancestro mas proximo.
42 abbr Representa una abreviacion o un acronimo ; la expansion de la abreviatura puede ser representada por el atributo title.
43 data Asocia un equivalente legible por maquina a sus contenidos. (Este elemento esta solamente en la version de la WHATWG del estandar HTML, y no en la version de la W3C de HTML5).
44 time Representa un valor de fecha y hora; el equivalente legible por maquina puede ser representado en el atributo datetime.
45 code Representa un codigo de ordenador.
46 var Representa a una variable, es decir, una expresion matematica o contexto de programacion, un identificador que represente a una constante, un simbolo que identifica una cantidad fisica, un parametro de una funcion o un marcador de posicion en prosa.
47 samp Representa la salida de un programa o un ordenador.
48 kbd Representa la entrada de usuario, por lo general desde un teclado, pero no necesariamente, este puede representar otras formas de entrada de usuario, como comandos de voz transcritos.
49 sub Representan un subindice.
50 sup Representan un superindice.
51 i Representa un texto en una voz o estado de animo alterno, o por lo menos de diferente calidad, como una designacion taxonomica, un termino tecnico, una frase idiomatica, un pensamiento o el nombre de un barco.
52 b Representa un texto hacia el cual se llama la atencion para propositos utilitarios. No confiere ninguna importancia adicional y no implica una voz alterna.
53 u Representa una anotacion no textual sin-articular, como etiquetar un texto como mal escrito o etiquetar un nombre propio en texto en chino.
54 mark Representa texto resaltado con propositos de referencia, es decir por su relevancia en otro contexto.
55 ruby Representa contenidos a ser marcados con anotaciones ruby, recorridos cortos de texto presentados junto al texto. Estos son utilizados con regularidad en conjunto a lenguajes de Asia del Este, donde las anotaciones actuan como una guia para la pronunciacion, como el furigana japones.
56 rt Representa el texto de una anotacion ruby.
57 rp Representa los parentesis alrededor de una anotacion ruby, usada para mostrar la anotacion de manera alterna por los navegadores que no soporten despliegue estandar para las anotaciones.
58 bdi Representa un texto que debe ser aislado de sus alrededores para el formateado bidireccional del texto. Permite incrustar un fragmento de texto con una direccionalidad diferente o desconocida.
59 bdo Representa la direccionalidad de sus descendientes con el fin de anular de forma explicita al algoritmo bidireccional Unicode.
60 span Representa texto sin un significado especifico. Este debe ser usado cuando ningun otro elemento semantico le confiere un significado adecuado, en cuyo caso, provendra de atributos globales como class, lang, o dir.
61 br Representa un salto de linea.
62 wbr Representa una oportunidad de salto de linea, es decir, un punto sugerido de envoltura donde el texto de multiples lineas puede ser dividido para mejorar su legibilidad.
63 ins Define una adicion en el documento.
64 del Define una remocion del documento.
65 img Representa una imagen.
66 iframe Representa un contexto anidado de navegacion, es decir, un documento HTML embebido.
67 embed Representa un punto de integracion para una aplicacion o contenido interactivo externo que por lo general no es HTML.
68 object Representa un recurso externo, que sera tratado como una imagen, un sub-documento HTML o un recurso externo a ser procesado por un plugin.
69 param Define parametros para el uso por los plugins invocados por los elementos object.
70 video Representa un video, y sus archivos de audio y capciones asociadas, con la interfaz necesaria para reproducirlos.
71 audio Representa un sonido o stream de audio.
72 source Permite a autores especificar recursos multimedia alternativos para los elementos multimedia como video o audio.
73 track Permite a autores especificar una pista de texto temporizado para elementos multimedia como video o audio.
74 canvas Representa un area de mapa de bits en el que se pueden utilizar scripts para renderizar graficos como graficas, graficas de juegos o cualquier imagen visual al vuelo.
75 map En conjunto con , define un mapa de imagen.
76 area En conjunto con , define un mapa de imagen.
77 svg Define una imagen vectorial embebida.
78 math Define una formula matematica.
79 table Representa datos con mas de una dimension.
80 caption Representa el titulo de una tabla.
81 colgroup Representa un conjunto de una o mas columnas de una tabla.
82 col Representa una columna de una tabla.
83 tbody Representa el bloque de filas que describen los datos concretos de una tabla.
84 thead Representa el bloque de filas que describen las etiquetas de columna de una tabla.
85 tfoot Representa los bloques de filas que describen los resumenes de columna de una tabla.
86 tr Representa una fila de celdas en una tabla.
87 td Representa una celda de datos en una tabla.
88 th Representa una celda encabezado en una tabla.
89 form Representa un formulario, consistiendo de controles que puede ser enviado a un servidor para procesamiento.
90 fieldset Representa un conjunto de controles.
91 legend Representa el titulo de un fieldset.
92 label Representa el titulo de un control de formulario.
93 input Representa un campo de datos escrito que permite al usuario editar los datos.
94 button Representa un boton.
95 select Representa un control que permite la seleccion entre un conjunto de opciones.
96 datalist Representa un conjunto de opciones predefinidas para otros controles.
97 optgroup Representa un conjunto de opciones, agrupadas logicamente.
98 option Representa una opcion en un elemento select, o una sugerencia de un elemento datalist.
99 textarea Representa un control de edicion de texto multilinea.
100 keygen Representa un control de par generador de llaves.
101 output Representa el resultado de un calculo.
102 progress Representa el progreso de finalizacion de una tarea.
103 meter Representa la medida escalar (o el valor fraccionario) dentro de un rango conocido.
104 details Representa un widget desde el que un usuario puede obtener informacion o controles adicionales.
105 summary Representa un resumen, titulo o leyenda para un elemento details dado.
106 command Representa un comando que un usuario puede invocar.
107 menu Representa una lista de comandos .
